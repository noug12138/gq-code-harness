gq-code-harness — 架构与流程

AI 自我进化引擎：踩过的坑变成机器能跑的检查，hook 在真实改动上硬拦——硬强制、不靠 AI 自觉。  |  跨平台 Node 插件 · v0.6.0

🔧 插件 = 机制（装一次 · 可复用 · 跨平台 · 自带 git/CI）

Hooks: session-inject · check-runner · doctor · stop-checks · gen-index Commands: init · new-task · archive · distill · contract · override Skill: harness-lifecycle Workflow: init-bootstrap（多 agent） lib/ · bin/ · assets/

📦 项目 = 内容（每个项目各一份 · 不进插件）

docs/ 长期知识 docs/harness/checks.json（弹药：可跑检查） .ai/ 任务与会话 CLAUDE.md 路由

插件不碰业务，只在项目内容上"跑检查 · 拦截 · 注入 · 起草"。

① 会话生命周期 —— 钩子时间线（Claude Code 直接执行）
SessionStart
→
session-inject注入 .ai/session.md，快速续上
→
干活中 · Edit/Write
→
check-runner · PreToolUse对"将写入的内容"跑快检查
→
命中检查 → DENY 硬拦改不进去；否则放行
→
收工 · Stop
→
doctor死链一致性 · 报告
stop-checks慢检查 + 做完没收 → exit 2 拦
gen-index按真实文件重算带标记索引
② 自我进化闭环 —— 核心：坑 → 可跑检查 → 硬拦 → 不复发
踩坑（三种硬信号）① 已有检查没过 ② 用户当场纠正 ③ 编译/测试失败
→
/contract带你写出可跑检查，坏例必抓 ✓ 好例不伤 ✓
→
checks.json这个坑的"弹药"
→
引擎执行PreToolUse 硬拦 + Stop gate
→
同坑不可能再靠"忘了"复发每犯一次没见过的错，系统多长一颗牙 ↺ 新坑回到起点

安全阀：/override（只有用户能运行，AI 不能自我放行，留痕） · 任何 hook 异常 fail-open（放行 + 报错，绝不锁死）。

③ 任务生命周期 —— 单文件 + status 字段；归档=翻字段、不搬目录
/new-task
→
.ai/tasks/<date>-x.mdstatus: active
→
干活 · 勾步骤
→
步骤全勾但仍 active → stop-checks exit 2 拦"做完没收"
→
agent 自己 /archivestatus→done，文件原地不动
→
/distill长期结论沉淀进 docs/

防退化靠机器把关、agent 自维护——不靠"记得归档"。

④ /init 多 agent 铺底 —— 一键给新项目铺底（确定性 JS 编排 + subagent 扇出）
/init
→
铺骨架 + 探测子系统scaffold assets · 10–16 个单元
→
并行测绘 agents（扇出）各读一个子系统，只回压缩小结，重读留各自即弃上下文
→
critic 自检查漏绕回补扫，保护上限 ≤2 轮
→
writer 起草落盘全标"机器起草·待人审"
→
汇总 ≤7 问题 → 一次性问你

编排器是确定性 JS（不碰文件系统）：读由测绘 agent、写由 writer agent，编排只倒腾压缩小结 → 主上下文不爆。扇出是为"规模/独立"，不是角色扮演（PM/前端/后端…）。
