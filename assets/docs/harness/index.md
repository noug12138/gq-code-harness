## 3. 什么时候新建
- 同一根因重复出现，或用户明确"别再犯"。
- 已有 grep / 测试 / CI 能判，但缺文档说明为什么卡。

## 4. 状态
- `active`：当前生效。
- `retired`：已被代码 / CI / 规则接管或场景消失，留档。

## 5. 复发与升级
- 同根因再发 → 对应 Cxxx 的 §5 追加一行。
- 可机器判定的部分 → `/gq-code-harness:contract` 写成可跑检查，进引擎硬拦。
